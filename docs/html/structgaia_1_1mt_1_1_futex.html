Detailed Description

An implementation of a simple futex (fast userspace mutex). Only wait and wake are implemented.

The main advantage of futex is performance. It avoids kernel involvement in uncontended cases. When thereâ€™s no contention, futexes allow threads to lock and unlock in userspace without entering the kernel, making operations significantly faster and reducing context-switch overhead. Only when there is contention does a futex use the kernel to put threads to sleep and wake them up, resulting in a hybrid model that is more efficient than mutexes, which always require kernel calls.
